If your initialization is failing, follow these steps to diagnose and fix the issue:

1. Check Your Google Apps Script Deployment

Make sure you've deployed your script correctly:

  • Click "Deploy" > "New deployment" in the Script Editor
  • Select "Web app" as the type
  • Set "Execute as" to "Me"
  • Set "Who has access" to "Anyone"
  • Copy the web app URL exactly

2. Verify Your Spreadsheet ID

Ensure you've replaced the placeholder spreadsheet ID with your actual Google Sheet ID:

// In your Google Apps Script code, find this line:
const SPREADSHEET_ID = '1LFshNLxJiTiBNZyQAmjtzuOfFkgRfANEIkKi5VDAcio';

// Make sure it contains your actual spreadsheet ID, not the example one

3. Check Script Permissions

Your script needs permission to access your spreadsheet:

  • Open your Google Sheet
  • Click "Share" in the top-right corner
  • Add the email address of your Google account (or service account) with edit permissions
  • If using a service account, share the spreadsheet with the service account email

5. Manual Initialization

If automatic initialization continues to fail, you can manually create the required sheets:

  1. Open your Google Sheet
  2. Create three sheets named: "Members", "Attendance", and "Settings"
  3. In the "Members" sheet, add these headers in row 1: ID, Name, Phone, Voice Part, Email, Registered Date, Status
  4. In the "Attendance" sheet, add these headers in row 1: ID, Member ID, Member Name, Date, Service Type, Status, Timestamp
  5. In the "Settings" sheet, add these headers in row 1: Key, Value
  6. Add these settings in the "Settings" sheet: adminPassword | admin123
    nextMemberId | 1
    nextAttendanceId | 1

6. Common Error Messages

Here are solutions for common error messages:

  • "Exception: Cannot call SpreadsheetApp.openById" - Your spreadsheet ID is incorrect or you don't have permission to access it
  • "Exception: You do not have permission to call SpreadsheetApp.openById" - Check your script permissions and spreadsheet sharing settings
  • "Exception: Service invoked too many times" - Google Apps Script has quotas. Wait a while and try again
  • "Exception: Unexpected error while getting the method or property openById" - Your spreadsheet ID format is incorrect
